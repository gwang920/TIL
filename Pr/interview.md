## EPC

```
- EPC란 ▲설계(Engineering) ▲조달(Procurement) ▲시공(Construction)을 의미합니다.
프로젝트 사업자가 설계부터 부품 조달, 시공, 시운전, 인도 까지 일괄 공급하는 것으로
일괄수주 방식과 같은 의미다.

- EPC의 핵심 쟁점

 '예산과 일정 관리' 입니다.
 예산은 계약주로 부터받기로한 계약한 금액을 말하는데 플랜트 건설을 하다 보면 예산 보다 더 들어가는 일이 태반입니다. 계약금 대비 얼마나 이윤을 남기느냐는 바로 실적으로 직결됩니다.

일정관리는 지체보상금과 같은 손실과 직결 -  계약 일 대비 늦춰지는 일 만큼 계약금에서 % 로 금액을 까 나가는 형태으로 손실이 발생

 준공일정, 협력사의 기자재 납품일정 등등 성공적인 EPC를 통해서 플랜트를 건설하고 난 후에는 사업주의 확인을 받습니다. 시운전을 통해서 전체 설비에 문제가 없는지를 점검하고 계약주가 OK 하면 EPC 업무가 끝나게 됩니다.
```

### 건설업

```
- 창업 2년 만에 유니콘기업으로 성장한 미국의 카테라는 새로운 생산체계를 제시했다는 평가를 받는다. 공장 생산 방식을 도입해 건축물 구성 요소를 모듈화(Modulization)하고, 이를 현장에서 조립한다. ‘현장 건설(site-build)’을 ‘현장 외 건설(off-site construction)’로 바꾼 것이다.

- 공장 생산은 글로벌 자재 공급망을 구축해 고품질 제품을 직접 수급하고, 생산에 활용함으로써 중개 과정의 비효율성을 제거한다. 또한, 표준화된 제품으로 품질관리 및 정확성을 높인다. 제작공장뿐 아니라 물류창고도 보유하고 있어 물류기업으로도 거듭나고 있다.
```

```
1) 설계
– 디지털 협업은 BIM, 클라우드 환경에서 사업 참여자들이 효율적인 3D 통합 설계 모델을 구축하는

사항 등을 (반)자동으로 체크하여 설계 오류를 최소화함.
```

## Hadoop

```
HDFS
- 다수의 리눅스 서버에 설치되어 운영되며 페타 바이트 이상의 대용량 데이터의 공간을 확보할 수 있습니다.
- 대규모 데이터를 분산, 저장, 관리하기 위한 분산 파일 시스템으로서, 저장하고자 하는 파일을 블록 단위로 나누어 분산된 서버에 저장됩니다.
- 배치작업에 적합하도록 설계되어 있으며, 빠른 처리보다 높은 데이터 처리량에 중점을 두고 있습니다.

맵리듀스 (MapReduce)
- 대용량 데이터를 분산 처리하기 위한 목적으로 개발된 프로그래밍 모델이자 소프트웨어 프레임워크 입니다..
- 맵리듀스 프로그래밍 모델은 사용자가 Map과 Reduce 두 함수를 이용하여 데이터를 처리하도록 합니다.
- 현재 MapReduce는 Java와 C++, 그리고 기타 언어에서 적용이 가능하도록 작성되었습니다.



- 데이터 마이닝(Data Mining) : 다량의 데이터 집합으로 부터 유용한 데이터, 즉 정보를 추출해내는 기법
- 빅데이터(Big Data) : 기존의 데이터베이스 관리도구의 데이터 수집/저장/관리/분석의 역량을 넘어서는 대량의 정형 또는 비정형 데이터셋
- HDFS(Hadoop Distributed File System) : 하둡 분산 파일 시스템으로, 데이터를 저장합니다.
- Hadoo MapReduce :  빅데이터 처리를 위한 프로그래밍 모델로 HDFS에 저장된 데이터를 처리한다.
- HBase : 분산 데이터베이스
- Mahout : 분산환경에서 처리할 수 있는 기계학습 알고리즘
```

### HTTP

```
HTTP - 프로토콜로 하나의 약속을 의미한다.

EX) 하나의 서버가 있고 다양한 브라우저들이 요청을 보내는데
	각 브라우저마다 다른언어로 요청을 보낸다면 서버는 그 요청을 받아 들이지 못한다.
	이를 위해서 하나의 약속인 HTTP를 규정한다.

규정된 약속,표준,프로토콜을 지켜 웹서버에 요청을 한다.
즉, HTTP는 서버(웹)과 클라이언트(PC,모바일 등의 웹브라우저)가 대화하기 위한 약속이다.
```





### POST, GET 방식

```javascript

 두 방식은 HTTP 프로토콜을 이용해서 서버에 무언가를 요청할 때 사용하는 방식.
 
Get : URL(주소창)을 통해서 모든 파라미터를 전달함.
	=> 주소창에 모든 전달 값이 노출되기에
		1) 보안의 문제를 갖는 파라미터(ID/PW)는 전달하면 위험
		2) URL길이의 제한이 있기에 많은양 전달 불가.

POST : 전달하고자 하는 정보가 HTTP Body에 포함되어 전달 됨.
	=> 사용자 눈에 노출되지 않고, 길이 제한 또한 없다.
	
그럼 언제 사용하나 ?
 GET : 	url을 사용하여 서버에서 정보를 가져오는 방식(ex.게시판 보기)
		가져오는것, SELECT적인 성향이 있음
 		서버에서 어떤 데이터를 가져와 보여주는 용도
 POST : 사용자로부터 정보를 받아 서버에 입력하거나 수정하는 방식(ex.게시판 글쓰기)

		수행하는 것, 서버의 값이나 상태를 바꾸기 위해 사용.

```
