# 도커

## 도커 컨테이너를 사용하는 이유

```
수평적 확장이 자유롭나요? 
서버의 견고함을 보장하면서도 동적으로 바꿀 수 있는 유연함이 존재하나요? 
퇴사를 하거나 부서를 옮겨야 해서 다른 이에게 서버 운영 기록을 인계하려면 시간이 얼마나 걸릴까요?

도커는 이에 대한 해답을 제시한다.
```

## 도커란 ?

```
컨테이너는 애플리케이션 개발환경에 구애받지 않고 실행할 수 있는 기술이다.
컨테이너 도구인 도커가 설치되어있으면
다양한 환경(우분투,centOS 등)이 각각 유일한 명령어를 갖는것과 달리 달리 통일된 명령어를 사용해
여러 프로그램을 설치하고 운영할 수 있다.
즉, 편리함과 확장성을 지니고 운영체제별로 존재하는 복잡한 설치과정을 거치지 않아도된다.
```



## 도커 이미지가 지닌 장점

### 1. 일관성

```javascript
"도커 이미지를 통해 일관성있게 프로그램을 관리하고, 동일한 환경을 쉽게 재현할 수 있다."

우선, 도커를 사용하지 않았을 때 발생할 수 있는 문제를 살펴보자.

한가지 예를 들면,
A라는 서버는 수개월전 구성했고, B라는 서버를 이제 막 구동하기 시작했다고 가정하자.
그렇다면, 두 서버에는 운영체제부터, 컴파일러, 설치된 패키지까지 차이를 가질 것이다.
그리고 이러한 차이가 장애를 일으키게 된다.
(A서버는 죽었는데 B서버는 잘되네?)와 같은 상황이 벌어질 가능성이 높다.

이러한 상황을 개선하기 위해 다양한 방식으로 서버 운영 기록을 저장해둔다.
그럼에도 불구하고 위와 같은 문제는 자주 발생하게 된다.



도커 파일 = 서버 운영 기록 코드화
도커 이미지 = 도커 파일 + 실행 시점
		  = 컨테이너를 실행할 수 있는 실행파일, 설정 값을 가지고 있는 것

도커 파일과 도커 이미지를 위와 같이 간단하게 정의할 수 있다.
도커파일이 '서버 운용 기록'이라면, 도커 이미지는 운영 기록을 '실행할 시점'인 것이다.

이러한 도커 이미지를 통해 일관성있게 프로그램을 관리하고, 동일한 환경을 쉽게 재현할 수 있다.
```

### 2. 실행속도, 성능

````
"실행속도와 성능이 다른 가상화 시스템에 비해 뛰어나다."


위 1번은 다른 가상화 시스템이 갖고 있는 장점이다.
하지만, 도커가 다른 가상화 시스템과 차이를 갖는 것은
실행속도와 성능에 있다.

가령 가상화 시스템은 가상화를 할 때 하드웨어 즉, cpu,메모리,하드디스크 등의 하드웨어 요소들을 가상화한다.
그렇기 때문에 속도, 성능 측면에서 도커에 밀리게 된다.
````
