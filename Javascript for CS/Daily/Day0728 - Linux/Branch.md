# Branch

```javascript
Branch는 단어 그대로 나뭇가지를 뜻한다.
즉, 하나의 중심에서 뻣어나가는 나뭇가지를 브랜치라고 볼 수 잇다.


브랜치란 "독립적으로 어떤 작업을 진행하기 위한 개념"이다.
각각의 브랜치는 다른 브랜치의 영향을 받지 않기 때문에 여러 작업을 동시에 진행할 수 있다.

쉽게 말해 개발자들이 동시에 다양한 작업을 할 수 있도록 하는 것이다.
버그를 수정하는 개발자, 새로운 기능을 만들어내는 개발자 등
동일한 소스코드에 다양한 작업이 동시에 이루어지며, 각각 서로 다른 버전으로 코드가 만들어 질 수 밖에없다. 
이러한 상황속에서 독립적으로 개발자들이 다양한 작업을 가능하게 하는 것이 브랜치이다.

그리고 이렇게 만들어진 브랜치는 다른 브랜치와 병합함으로써 작업한 내용을 다시 새로운 하나의 브랜치로 모을 수 있다.

각자 작업을 진행한 후, 작업이 끝난 사람은 메인 브랜치에 자신의 브랜치의 변경 사항을 적용한다. 
이렇게 함으로써 "다른 사람의 작업에 영향을 받지 않고 독립적으로 특정 작업을 수행"하고 
그 결과를 하나로 모아 나가게 된다. 이러한 방식으로 작업할 경우 '작업 단위', 
즉 브랜치로 그 "작업의 기록을 중간 중간에 남기게 되므로" 문제가 발생했을 경우 
원인이 되는 작업을 찾아내거나 그에 따른 대책을 세우기 쉬워진다.


브랜치를 사용하는 이유? (From BoostCampers)
    
1) 병합한다는게 무엇을 병합한다는걸까요? git을 사용한다면 branch를 병합하는것이고 
git을 사용하지 않는다면 파일을 통째로 병합해야하는데 이때 생기는 문제에 대해 생각해보시면 될 것 같습니다

2) master에는 실제로 실행 중인 서비스
develop에는 개발중인 코드
기능별로 branch를 나누는 등
조금 더 업무적으로 분할되니 좋은거 같습니다.
거기다 branch를 따로 나누지 않고 master에 작업하면 타인이 push하면 
제가 push 할 때 지속적으로 pull&merge를 해줘야 해서 불편합니다.

3) master branch를 항상 배포 가능한 상태로 유지하기 위해 branch와 merge를 사용한다.
```
