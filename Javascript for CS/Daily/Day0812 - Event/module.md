# 모듈화

```
코드를 모듈화하기 위한 기준으로 featuer를 잡고 싶다.
feature에 따라 모듈화하는 것은
말 그대로 각 기능에 따라 코드를 모듈화하는 것이다.

기능에 따라 나누면 코드를 재사용하기 쉽고,
모듈의 협력관계를 보다 유연하게 설계 할 수 있다고 생각한다.

그리고 함수는 단 한 가지의 일을
클래스는 단 하나의 책임을 갖게하면 더 좋을 것 같다.
```

```
예를 들어 커피숍을 구현한다하자.

이때, 커피숍에서 이루어지는 모든 행위와 존재하는 모든 객체들은 서로 모듈화 되어야할 것이다.
커피숍은 커피메뉴판,주문 리스트, 바리스타, 매니저, 계산원, 손님, 커피객체, 주문현황판 정도로 모듈화할 수 있을 것 같다.

이에 따라

커피메뉴판       			  주문현황판
 ^				  				^
 |                				|
 v                				v
손님 <-> 계산원 <->주문리스트 <-> 매니저 <-> 바리스타 <-> 커피객체


1) 손님은 커피 메뉴판을 확인한다.
2) 손님은 계산원에게 주문한다.
3) 계산원은 주문 리스트를 만든다.
4) 매니저는 주문 리스트를 확인한다.
5) 매니저는 주문현황판을 갱신한다.
6) 매니저는 바리스타에게 커피 주문을 전달한다.
7) 바리스타는 커피객체로부터 커피를 만든다.
8) 바리스타는 커피를 매니저에게 전달한다.(주문현황판을 갱신한다.)

이처럼 모듈화된 객체들의 협력관계를 일률적으로 설정한다면 보다 효율적인 코드 구성이 가능할 것 같다.
```
