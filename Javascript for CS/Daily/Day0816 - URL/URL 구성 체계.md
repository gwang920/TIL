# URL 구성 체계

```
URL의 구조는 프로토콜 식별자와 자원이름으로 구성되어있다.
자원이름은 호스트주소와 포트번호로 구성된다.

일반적인 URL의 체계는 다음과 같다.

프로토콜://호스트주소:포트번호/파일#섹션
```

![image](https://user-images.githubusercontent.com/49560745/90644255-7b885180-e26f-11ea-9425-11767e03b28e.png)

```xml
1. 프로토콜

 http://

통신 방식을 미리정해놓은 것이다.
네트워크상으로부터 웹 문서(html)나 이미지 등의 리소스를 받아오는 방식을 정하는 것.
http, https 등이 있다.

2 ~ 5. 호스트

 www.codns.com

통상적으로 2,3,4,5를 호스트라 부른다.
호스트는 ip 주소나 mac주소처럼 기계적인 이름을 대신해 읽기 쉽도록 만든 것이다.

6. 포트번호

  80

포트번호는 네트워크 또는 인터넷을 사용하여 통신하는 각 응용프로그램 또는 프로세스의 논리주소이다.
HTTP의 기본포트는 80으로 생략이 가능하다.

7 ~ 8. 경로

codns/codns.jsp

해당 파일이 존재하는 경로를 말한다.
파일은 상위 폴더, 또 그 상위 폴더를 거쳐서 들어가는 방식이기에 루트 디렉토리부터 파일 위치까지
경로를 표현한다.

10. 쿼리 스트링

 ?id=1
 
 ?뒤에 존재하는 키,값 쌍의 모음을 쿼리스트링이라고한다.
 ?를 기점으로 url과 쿼리스트링을 구분한다.
 
 위 url을 기준으로하면 codns.jsp를 실행할 때 id=1인 값을 가지고 실행하라는 의미이다.
 
 
 만약 codns.jsp 파일에 다음과 같은 코드가 있다면
----------------------------------------------------------------------------------------- 
 id는 <b><%=request.getParameter(id)%></b>입니다.;
-----------------------------------------------------------------------------------------

html 출력
-----------------------------------------------------------------------------------------
 'id는 1입니다.'
-----------------------------------------------------------------------------------------
 가 출력되는 것이다.
```
