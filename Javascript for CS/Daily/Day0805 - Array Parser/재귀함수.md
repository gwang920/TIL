# 재귀함수

```
장점
- 코드를 한 눈에 알아보기 쉽다.
- 알고리즘 자체가 재귀적으로 표현하기 자연스러울 때 사용하기에 표현의 장점(?)이 있는 것 같다.
- 변수 사용을 줄여준다.


단점
- 흐름을 따라가기 쉽지 않다.
- 메모리상의 이점이 크지 않다.

재귀 함수를 사용하면 함수의 호출이 스택에 차곡 차곡 쌓이게 되고, 
위에서부터 차례대로 값을 반환하기 전에는 계속 메모리 공간을 차지한다.
그렇기 때문에 호출 스택이 너무 커져서 메모리를 엄청나게 소비할 수도 있다.
```

# 재귀함수 동작방식

```javascript
재귀함수는 호출되면 스택에 쌓이게 된다.

-----------------------------------------------------------------------------------------
function f(n) 
{
 if(n==1) return 1;
 return n + f(n-1);
}
-----------------------------------------------------------------------------------------

f(5)의 동작방식을 살펴보면
우선 f(5)가 호출되면 스택에 f(5)가 쌓이고, 
    
-----
f(5)
stack


f(4)를 호출한다. 이때 또 다시 f(4)는 스택에 쌓이게 된다.
-----
f(4)
f(5)
stack

마찬가지로 f(3)가 호출되고 f(3)는 스택에 쌓인다.

.
.
.

f(1)이 되었을 때 비로소 f(1)은 값을 반환하며 메모리에서 해제된다.
-----
f(2)
f(3)
f(4)
f(5)
stack

이에 따라 순차적으로 f(2) -> f(3) -> f(4) -> f(5)가 해제된다.
즉, 스택의 동작방식과 일치한다.

```

# 재귀함수 무한

```
재귀함수는 자신을 다시 호출하면서 새롭게 변수들이 생성된다. 이에 따라 메모리 문제가 발생하게 된다.
=>(재귀함수는 부분 문제가 해결되지 않으면 상위 문제들도 계속 풀리지 않는다.)
만약 무한에 가까운 재귀함수를 돌리면 엄청난 메모리 낭비가 발생하며 스택 오버플로우가 발생한다.
=>(스택은 유한하기에 무한정 담을 수 없다.)

```
