# 내부 ip vs 공인 ip

```ㅇ
내부 ip(=사설 ip)
외부 ip(=공인 ip)

우선 공인 ip는 신뢰할 수 있는 기관으로부터 공인받은 주소이다.
전 세계에서 유일한 값을 지니며 인터넷 세상에서 데이터를 주고 받는
'진짜 통로'이다.

사설 ip는 공인 ip의 단점을 커버하기 위해 새롭게 ip를 가상으로 할당하는 것이다.
공인 ip의 개수는 약 42억개로 이미 정해져있고, 이론적으로 사용할 수 있는 개수가 한정적이다.
앞으로 인터넷 사용 디바이스는 급격하게 증가할 것이고
머지않아 공인 ip는 모두 소비될 것이다.

그렇기에 공인 ip에 공유기를 설치해 여러 사설 ip를 할당해 사용해 이러한 문제점을 해결하는 것이다.

                  공인ip 220.72.101.25
                  	|
                  	|
                  	v
                  공유기 191.168.0.1
                    |
		 -----------|-------------
         |          |            |
         |          |            |
	     v	        v            v
사설ip 192.168.0.2   192.168.0.3   192.168.0.4
```





```
사설 ip로 웹서버를 구축하기 위해
포트포워딩을 사용할 수 있다.

* 포트포워딩은 포트(port)를 전달(forwarding)해주는 작업이다.


예를 들어 회사에서 부여받은 공인 ip를 219.214.33.35 라고 가정하자
그리고 세 대의 컴퓨터 192.168.0.3 192.168.0.4 192.168.0.5를 사용한다고 가정하자.

이때 공유기가 공인 ip 219.214.33.35의 80번 포트로 들어오는 패킷을 192.168.0.5으로 보내준다.

이렇게 되면 192.168.0.5 컴퓨터를 웹 컴퓨터로 사용할 수 있는 것이다.
이처럼 특정 포트로 들어오는 패킷을 내부 컴퓨터로 넘겨주는 것을 포트포워딩 이라한다!

```
