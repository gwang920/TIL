# 데이터분석

```
- 데이터를 산출하고 분석하여 효과적으로 이용하기 위한 필수 요건은 최대한 핵심만 뽑아 간결하게 전달하는 것
- 데이터, 하드웨어 그리고 소프트웨어는 언제나 함께 묶어서 생각해야 한다.
```



##### 1. 분석이 필요한 내용을 식별하고 구조화하기

````
적절한 정량적 분석은 문제를 식별하거나 분석한 내용을 결정하고 이것을 해결하려는 시도를 하는 것에서부터 출발



분석할 내용이 무엇인지 결정하는 단계 - '구조화하기(framing)'

문제를 바라보는 사람들의 일반적인 감각을 말하는 단순한 호기심, 일에서의 경험 또는 결정을 하거나 행동을 하기 위한 필요사항 등 다양한 요소

 

이 초기 단계에서 분석은 아직 진행되지 않습니다. 결정을 이끌어 낼 수 있게 하는 주요 분석 내용은 어쩌면 직감이나 직관으로 인해 나올 수도 있습니다. 이때 분석 내용을 뒷받침하는 근거는 다소 부족할 수 있습니다. 물론 전반적인 정량적 분석의 요점은 당신의 데이터에 대한 직관 능력을 테스트하는 것일 수도 있습니다. 분석적인 사람들은 자신의 직감과 직관을 데이터 분석을 통해 맞는 것인지 확인하는 경향이 있는데, 이것이 분석적인 사람들과 아닌 사람들의 차이를 만드는 주요한 요소라고 할 수 있습니다.


````



##### 2. 정량적인 분석을 하는 사람들과 일하기

```
정량적 분석을 이야기함에 있어 관리자가 함께 일하는 팀과 긴밀한 관계를 맺는 것은 아주 중요한 부분입니다. 당신이 비즈니스에서 나타나는 문제를 잘 이해하고 있다는 것은 즉, 당신의 '수리(quant)’ 능력이 어떻게 데이터를 모으고 분석할 수 있는지를 이해한다는 것을 의미합니다. 다른 이들과 관계를 긴밀하게 맺으며 일을 하기 위해서는 팀 내의 모든 사람들이 서로 스스럼없이 다가가 이야기를 할 수 있어야 합니다. 당신이 만약 정량적인 분석을 하지 않는 관리자라면 주로 정량적은 분석을 하는 팀원들이 그들이 어떤 부분을 문제로 인식하고 해결해야 하는지 도움을 주어야 합니다. 그러면 그들은 며칠 내로 그들이 필요한 부분을 찾고 일을 하기 시작할 것입니다. 당신의 수리 능력은 일반적인 비즈니스 용어들 사이에서 데이터 분석을 담당하는 직원들과 커뮤니케이션을 하는 수단으로서, 그리고 문제를 인식하고 당신이 만족할만한 정도까지 일을 하는데 필요한 요소입니다. 만약 당신의 분석가 팀원이 당신과 마주하여 이야기하는 것을 꺼려한다면 당신은 정량적인 분석에 아직 익숙하지 않은 것일 수 있습니다. 그렇지만 당신은 그들과 일하면서 점점 익숙해질 것이고 마침내 합의점을 찾을 수 있을 것입니다.
```



##### 3. 다양한 종류의 데이터와 그것이 어떤 영향을 주는지 이해하기

```

스몰데이터 - 관리가 가능하지 않은 작은 데이터를 의미 
		 보통은 하나의 서버에 저장할 수 있을 정도의 정보
 
 ex) 회사의 결제 시스템, 예를 들어 재무 시스템에서 나오는 정보
 	회사의 재무 성과와 유통 구조를 보여준다.

 

빅데이터 - 하나의 서버에 넣기에는 너무 많은 정보이고 비교적 구조화가 안되어 있고 너무 빨리 변화한다. 비즈니스 밖에서 일어나는 데이터 

  ex) 고객들과 잠재고객들이 소셜 미디어에서 당신의 회사에 대해 뭐라고 말하는지, 
  그들이 콜센터의 직원과 뭐라고 대화하는지, 그리고 당신의 매장에서 얼마나 사람들이 많이 방문하는지 등에 대한 정보 
```



##### 4. 다양한 종류의 분석과 그것이 어떤 영향을 주는지 이해하기

````
예측 분석 - 미래를 예측하기 위해 과거의 통계 모델을 사용

바로 자동 분석(Automated Analytics) - 사람이 수행하는 것은 아니고 컴퓨터에 의해서 수행되는 것

은행의 신용장 발행, 보험 가입 대상 확인 등이 이 분석 시스템을 통해 완전 자동화될 수 있다.
````



##### 5.분석 결과를 내부적 또는 외부적 용도로 사용해야 하는지 구분하기

```
마지막으로 관리자들은 분석된 데이터를 내부적 또는 외부적으로 사용할 수 있도록 구분하는 작업을 해야합니다. 전통적으로 데이터 분석은 전적으로 내부적으로 결정을 내리기 위해서 이용되어 왔습니다. 이것은 여전히 내부적인 결정을 위해 중요하지만 이제는 회사들이 이 분석 정보를 새로운 제품과 서비스를 만드는데도 이용할 수 있습니다. 이는 당신이 생각하는 것처럼 구글, 링크드인 같은 디지털 관련 회사나 GE, 몬산토와 같은 대기업 그리고 데이터로 제품을 몇몇의 대형 은행에만 적용되는 내용은 아닙니다. 관리자들이 데이터 분석을 내부적, 외부적으로 적극적으로 이용하려 하는 한 이것은 회사에게는 새로운 옵션이 될 것입니다.
```
